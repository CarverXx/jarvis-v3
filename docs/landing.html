JARVIS v3

A voice assistant with two brains: a fast conversational LLM that speaks for you, and a tool-calling agent that actually runs the commands. Wake-word triggered, half-duplex, entirely on your machine.

View on GitHub → Install How it works

Why a dual brain?

Most voice assistants pipe everything through one model. That model has to be fast enough for small talk and smart enough to call tools, so it usually ends up being neither. JARVIS v3 splits the job.

Fast layer

Subconscious

Handles dialog, persona, simple facts, and the initial read on every turn. Stateful per-session with a sliding-window history. Decides in ~200 ms whether the turn needs real action.

If yes, it returns an OpenAI-format tool_calls[0] pointing at invoke_hermes(task=...) — and the second brain takes over.

Model · SGLang-served Qwen3.6-35B-A3B-FP8
Context · 128K · prefix-cached
TTFB · < 1 s for direct replies

Agent layer

Hermes · Main consciousness

An agentic loop that actually runs things. Multi-step reasoning, multi-tool chains, retries. It can read your Markdown wiki, search the web, list Cloudflare zones, list systemd services, write notes into an inbox, or read arbitrary files under your home.

Returns a plain-text result, which the subconscious turns back into a natural 2-sentence spoken reply.

Backend · Hermes Agent CLI (Nous Research)
Tools · MCP server · fs / web / rag / cf / system
Runtime · 5–30 s depending on depth

Architecture

Six services running on a single Linux box. Wake and VAD are CPU-only; the three models share one GPU.

Features

Design choices that make it usable day-to-day, not just a demo.

Tenet 01
Everything local by default

No OpenAI / Anthropic / Google calls. LLM, ASR, TTS all run on your GPU. The only network call is web.search, and only if you opt in by dropping a Tavily key in place.

Tenet 02
Zero-shot voice cloning

VoxCPM2 takes a 4–8 second reference clip and uses that voice for every response. One time setup; consistent character across days.

Tenet 03
Personalised wake word

Record 20 samples of yourself saying the wake word; train a custom verifier. False positives from TV / other voices drop, your own recall goes up.

Tenet 04
Echo-cancelled half-duplex

Mic hard-mutes during TTS, queue flushes on state transitions, wake cool-down on IDLE. No more "assistant says its own name on speaker → re-triggers itself forever" loop.

Tenet 05
Bring-your-own knowledge base

Point JARVIS at any folder of Markdown files. rag.search + fs.read become the agent's reading eyes. Your content, your categories, your schema.

Tenet 06
Live terminal dashboard

One command launches a six-panel rich.Live TUI: state machine, mic sparkline, wake scores, ASR transcript, streaming subconscious tokens, Hermes heartbeat, TTS progress.

What a turn looks like

Three canonical examples — the same pipeline, three different paths.

You sayRoutePath taken
"Hey Jarvis, 你好" direct Subconscious replies in ~300 ms. No tool call. TTS speaks.
"Hey Jarvis, 今天几号" tool · system.date Subconscious refuses to guess dates; calls invoke_hermes(task="…"). Hermes runs system.date, returns plain text. Subconscious says it.
"Hey Jarvis, 查一下我笔记里 X" agent loop Hermes calls rag.search("X"), gets a file path, follows up with fs.read(), summarises content. Subconscious turns the summary into a 2-sentence spoken answer.

Install

Roughly 30 minutes if you already have CUDA + Python 3.12. Run each command on the Linux host you want the assistant to live on.

  1. Clone and virtual-env
    git clone https://github.com/CarverXx/jarvis-v3 ~/jarvis-v3
python3.12 -m venv ~/jarvis-v3-venv
source ~/jarvis-v3-venv/bin/activate
pip install -r ~/jarvis-v3/requirements.txt
  2. Download the three models
    huggingface-cli download Qwen/Qwen3.6-35B-A3B-FP8
huggingface-cli download Qwen/Qwen3-ASR-1.7B
huggingface-cli download openbmb/VoxCPM2
  3. Install Hermes Agent CLI

    Point its config at your local SGLang endpoint.

    # ~/.hermes/config.yaml
model:
  default: Qwen/Qwen3.6-35B-A3B-FP8
  base_url: http://127.0.0.1:8000/v1
  4. Optional: API keys for web / Cloudflare tools
    mkdir -p ~/.config/jarvis && chmod 700 ~/.config/jarvis
echo "REPLACE_ME_TAVILY_KEY"       > ~/.config/jarvis/tavily.key
echo "REPLACE_ME_CLOUDFLARE_TOKEN" > ~/.config/jarvis/cf.token
chmod 600 ~/.config/jarvis/*.key ~/.config/jarvis/*.token
  5. Install systemd units

    Customize each .sample file (username, audio device, SGLang api-key) before installing.

    for f in ~/jarvis-v3/systemd/*.sample; do
    base="$(basename "$f" .sample)"
    sudo install -m 644 "$f" "/etc/systemd/system/$base"
done
sudo systemctl daemon-reload
sudo systemctl enable --now sglang mcp-server qwen3-asr-shim voxcpm2-tts hermes-shim jarvis-v3
  6. Launch the TUI dashboard
    ~/jarvis-v3-venv/bin/python ~/jarvis-v3/tui/dashboard.py

    Now say "Hey Jarvis" at the mic.

Customise further

Two environment variables let you swap the persona and voice-output rules without editing code:

Env varControls
JARVIS_SUBCONSCIOUS_PROMPT The subconscious persona — how the assistant talks in direct conversation. Override for different languages, tones, or escalation policies.
JARVIS_HERMES_VOICE_PROMPT The style rules applied to every Hermes tool-call output. Override to change output length / formality / what Hermes should strip from tool results before speaking.
MIC_PAREC_SOURCE PulseAudio source name for mic capture. Find via pactl list sources short | grep -v monitor.
WAKE_THRESHOLD openWakeWord score threshold. Default 0.5; with a custom verifier, 0.7 is fine.
