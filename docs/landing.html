HERMES AGENTIC RUN :8004 · optional
Subprocess ~/.local/bin/hermes chat -q against the same SGLang backend; tool-using loop with file / shell / net access. Result fed back to subconscious for a 1-2 sentence spoken summary.
event: hermes_start → tick(2s) → hermes_done · ≤ 120 s

06 · DESIGN TENETS · RISK · EVOLUTION

Design Tenets & Risk Register核心决策 · 风险清单 · 版本演进

// why we chose this architecture and what could still break it
TENET 01

Half-duplex over full-duplex

放弃 v2 的全双工 barge-in 调度。简单、可维护、家庭场景够用。

TENET 02

Single-gate wake

wake-score + VAD 一次判定，不做 ASR 二次确认 —— 业界最佳实践（openWakeWord / Amazon Science）。

TENET 03

Dual-agent brain

潜意识处理 80% 日常对话求快（TTFB < 1s），主意识 Hermes 按工具委托处理深度任务。

TENET 04

Software mic-mute

TTS 播放期间直接丢帧，不依赖 PipeWire AEC —— 可预测、可调试。

TENET 05

Local-only inference

零外发请求、所有模型跑在单机 RTX PRO 6000 上。隐私即架构。

TENET 06

systemd everywhere

每个服务独立进程 + Restart=on-failure + After= 依赖链，7×24 自愈。

TENET 07

Observability first

结构化 events.jsonl + TUI dashboard，每个事件都可事后回放。

TENET 08

Follow-up window

TTS 完播后 15s 免唤醒词窗口，模拟真人对话节奏。

TENET 09

Streaming by default

TTS chunked、subconscious token-stream、ASR 小音频批——所有能流式的都流。
